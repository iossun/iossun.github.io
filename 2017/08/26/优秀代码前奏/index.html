命名规范(尽量模仿OC系统命名)

属性、变量的命名
  1. 遵守小驼峰命名规则
  2. 变量的名称一般同时包含功能与类型

  3. 系统常用类作实例变量声明时加入后缀

    | 类型 | 缩写 |
    | —— | :—-: |
    | NSString | Str |
    | UIButton | Btn |
    | UILabel | Lbl |
    | UIImage | Img |

    | UIView | View |
    | UIViewController | VC |

类的命名

类名（以及类别、协议名）应首字母大写，并以驼峰格式分割单词。

  1. 类的前缀：所有类名、枚举、结构、protocol定义时最好加一个统一的标示符，可以是项目缩写，或者个人项目的名称缩写，例如都加上全大写的SM（我的姓氏）或 HW(华为公司名缩写)作为前缀
  2. 类的后缀：所有的控制器都加上Controller,所有的通知名都加上Notification，所有protocol定义时，都加上后缀Delegate
方法命名

  1. 方法名应遵守小驼峰原则，首字母小写，其他单词首字母大写,每个空格分割的名称以动词开头。执行性的方法应该以动词开头，小写字母开头，返回性的方法应该以返回的内容开头，（遵循见名之意）

  2. 代理方法:以发送代理的对象类名作为代理方法名的开始（去掉类名的前缀，并且小写开头）
